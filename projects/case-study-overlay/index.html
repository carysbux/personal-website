Problem

LLMs can see images users send, but not touch them with editing. AI is describing parts of an image based off of training data instead of information from the specific picture sent Lack of confirmation the user is given an unsure answer about a physical part and will act on it

Goal

Using Gemini and Bi-Directional Visual Grounding, we can create an app or web extension that can edit an image with simple elements and send it back to the user. AI should be able to "markup" the user’s original image in real-time to confirm its analysis and guide the user’s physical movements.

Solution

The main objective was to create a real-time tool that integrates Gemini's multimodal analysis to solve the spatial orientation gap. It can replace ambiguous dialogue with instant visual validation and draw simple graphics with the frontend, providing the user with a high confidence guide.

Features & Benefits

Feature Function Benefit
Active Overlay Gemini's AI generates a non-destructive SVG layer over the user's photo. Instant confirmation of component identification.
Guided Sequences Markups change dynamically as the user progresses through steps. Visual checklist for complex physical or digital problem solving.
Reference Pinning Users can drop a "pin" on a part and ask, "What is this?" Eliminates the need for "top-left" verbal descriptions.
